The Discovery Node exposes a RESTful JSON API for content search, teaser ingest, publisher management, node federation, statistics, and administration.

All API endpoints are prefixed with `/api/v1/` (main API) or `/api/web/v1/` (website-internal API).

## Interactive documentation

Swagger/OpenAPI documentation is available at:

```
http://localhost:3300/api/docs
```

The Swagger UI allows you to explore endpoints, view request/response schemas, and execute test requests directly from your browser.

## Request format

- **Content-Type**: `application/json` for all POST/PUT requests
- **Accept**: `application/json`
- All request bodies are validated using class-validator with `whitelist: true` and `forbidNonWhitelisted: true` — unknown properties are rejected

```bash
curl -X POST http://localhost:3300/api/v1/search \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{ "query": { "text": "climate policy" } }'
```

## Authentication headers

The API uses two types of API key authentication depending on the operation:

| Header | Purpose | Required for |
|--------|---------|-------------|
| `X-Publisher-Key` | Publisher API key (issued at registration) | Teaser ingest (publish, update, delete, batch) |
| `X-Admin-Key` | Admin API key (set via `ADMIN_API_KEY` env var) | Admin, publisher management (update, suspend, rotate-key), node management |

<Callout kind="info">
  Search, suggestions, health, statistics, and the public node directory do **not** require authentication. They are open endpoints available to any client.
</Callout>

## Pagination

Paginated endpoints use `from`/`size` offset-based pagination:

```json
{
  "pagination": {
    "from": 0,
    "size": 20
  }
}
```

Responses include pagination metadata:

```json
{
  "pagination": {
    "from": 0,
    "size": 20,
    "total_pages": 62,
    "has_next": true
  }
}
```

Maximum page size is **100** results.

## Error responses

All errors follow a consistent format:

```json
{
  "statusCode": 400,
  "message": "Teaser validation failed",
  "errors": ["identifier.content_id is required", "overview.headline is required"]
}
```

### HTTP status codes

| Code | Meaning |
|------|---------|
| `200` | Success |
| `201` | Created |
| `400` | Bad Request — validation failed or malformed input |
| `401` | Unauthorized — missing or invalid API key |
| `403` | Forbidden — publisher not authorized for this action |
| `404` | Not Found — resource does not exist |
| `409` | Conflict — duplicate resource (e.g. domain already registered) |
| `429` | Too Many Requests — rate limit exceeded |
| `500` | Internal Server Error |

## Rate limiting

The API enforces rate limits at two levels:

### Global rate limit

**120 requests per minute per IP address** across all endpoints, enforced by @nestjs/throttler.

When exceeded, the API returns:

```json
{
  "statusCode": 429,
  "message": "ThrottlerException: Too Many Requests"
}
```

### Per-publisher rate limit

Publisher ingest endpoints have additional per-publisher limits based on [verification level](/concepts#verification-levels), enforced via a Redis sliding window:

| Level | Teasers/day | Requests/min |
|-------|-------------|--------------|
| Unverified | 100 | 60 |
| Domain verified | 1,000 | 120 |
| Organization verified | 10,000 | 300 |
| Trusted publisher | 100,000 | 1,000 |

## CORS

CORS is configured via the `CORS_ORIGIN` environment variable:

- **Development**: `*` (allow all origins)
- **Production**: comma-separated list of allowed origins

Allowed methods: `GET`, `POST`, `PUT`, `DELETE`

Allowed headers: `Content-Type`, `Accept`, `X-Publisher-Key`, `X-Admin-Key`, `Authorization`

## Compression

All responses larger than 1 KB are automatically gzip-compressed.
