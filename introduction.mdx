The **roadbeat Discovery Node** is a distributed search and content discovery server that indexes structured content teasers from verified publishers across the roadbeat network. It provides faceted search, geographic filtering, goal-driven discovery, and semantic search — all without tracking users or optimizing for engagement.

<Callout kind="info">
  Discovery Nodes are the "search engines" of roadbeat. Unlike traditional search engines, they are decentralized, content-type aware, goal-aligned, and privacy-preserving.
</Callout>

## What a Discovery Node does

<Columns cols={3}>
  <Card title="Search & Filter" icon="search" href="/features/search">
    Full-text search with multilingual analyzers, faceted filtering, geographic queries, and transparent relevance scoring.
  </Card>

  <Card title="Content Ingest" icon="upload" href="/features/ingest">
    Accept and index content teasers from verified publishers with signature verification and automatic embedding generation.
  </Card>

  <Card title="Goal-Driven Discovery" icon="target" href="/features/search#goal-based-search">
    Match content to user goals using filterset templates with parameter substitution — no algorithmic feed.
  </Card>
</Columns>

<Columns cols={3}>
  <Card title="Publisher Registry" icon="building" href="/features/publishers">
    Register publishers with domain verification, trust levels, and per-publisher rate limits.
  </Card>

  <Card title="Semantic Search" icon="brain" href="/features/semantic-search">
    Optional AI-powered vector search using OpenAI embeddings for hybrid BM25 + kNN similarity scoring.
  </Card>

  <Card title="Transparency Website" icon="globe" href="/features/website">
    Public-facing Next.js dashboard showing node stats, publishers, content types, and health — in 24 EU languages.
  </Card>
</Columns>

## Tech stack

| Component | Technology |
|-----------|------------|
| **Runtime** | Node.js 20 LTS |
| **API Framework** | NestJS 10.x |
| **Search Engine** | Elasticsearch 8.11.4 |
| **Database** | PostgreSQL 18.x (Prisma ORM) |
| **Cache** | Redis 7 (ioredis) |
| **Message Bus** | NATS JetStream 2.10 |
| **Website** | Next.js 14 (App Router) |
| **i18n** | next-intl v4 (24 EU locales) |
| **Security** | Helmet, @nestjs/throttler, Ed25519 signatures |

## What Discovery Nodes store vs. don't store

<Tabs>
  <Tab title="Stored" icon="database">
    - **Teasers** — title, description, hero image, key metadata (not full content)
    - **Filter facets** — all searchable/filterable metadata fields
    - **Publisher info** — source identification, trust signals, verification status
    - **Temporal data** — publication dates, event dates, expiration dates
    - **Geographic data** — locations for geo-filtered queries
    - **Embeddings** — vector representations for semantic search (generated server-side)
  </Tab>
  <Tab title="Not Stored" icon="shield">
    - **Full content** — article bodies, full media, detailed data (stored at source)
    - **User data** — no user profiles, no consumption history
    - **Social graph** — no follower/following relationships
    - **Payment data** — no transaction or pricing information
    - **Behavioral data** — no tracking, no profiling, no engagement metrics
  </Tab>
</Tabs>
