<Update label="2026-03-14" description="v1.1.0" tags={["feature"]}>
## Batched goal search & random mode

**New endpoint: `POST /api/v1/search/by-goals`**
- Accepts an array of pre-resolved `content_queries` from one or many user goals and builds a **single Elasticsearch `bool.should` query** — enabling 150+ content_queries to execute in one ES request instead of 150 separate ones
- Each content_query becomes a `should` clause with content_type filter, classification filters, and relevance-based boost weighting (high=1.5, medium=1.0, low=0.5)
- Supports **named queries** for goal attribution — each clause carries `_name: "goal:{goalId}:cq:{index}"` so ES returns `matched_queries` identifying which goals matched each result
- Global user filters (date range, language, publisher, geographic) applied at the top-level `filter` clause across all goal queries
- Uses wildcard index `roadbeat-content-*` to avoid `index_not_found_exception` when some content types have no indexed content yet

**Random / Surprise mode**
- New `random` flag on the batch endpoint triggers `function_score` with `random_score` for serendipitous content discovery
- Session-stable pagination via `random_seed` — same seed produces same ordering within a session
- Optional light bias from user's goal categories (low boost 0.3) so random results lean slightly toward the user's interests

**Goal catalog filter mapping**
- `mapGoalCatalogFilters()` translates goal catalog filter paths to Elasticsearch classification paths: `category.tags` → `classification.tags`, `category.categories` → `classification.categories`, `category.primary` → `classification.primary_category`, `category.topics` → `classification.topics`
- DN-native paths (`classification.*`, `temporal.*`, `access.*`, `publisher.*`, `geographic.*`, `metadata.*`) passed through directly
- Unknown content-type-specific keys (e.g. `event_type`, `mood`, `industry`) are silently ignored since the teaser generator does not populate them in the index
</Update>

<Update label="2026-03-10" description="v1.0.0" tags={["feature"]}>
## Initial Release

The roadbeat Discovery Node v1.0.0 includes a complete distributed search and content discovery server with the following capabilities:

**Search & Discovery**
- Full-text BM25 search with German and English analyzers
- Hybrid search (BM25 + kNN vector similarity) via OpenAI embeddings
- Goal-based search with filterset template parameter substitution
- Autocomplete suggestions with phrase prefix and fuzzy matching
- 11 global facets + 4 content-type-specific facet sets

**Content Ingest**
- Teaser publish, update, delete, and batch operations (up to 100)
- Ed25519 signature verification for content integrity
- Automatic embedding generation at ingest time
- Per-content-type Elasticsearch indices

**Publisher Management**
- Publisher registration with one-time API key
- DNS TXT record domain verification
- 4-tier verification levels with tiered rate limits
- Admin operations: key rotation, suspension, updates

**Node Federation**
- Peer node directory with heartbeat protocol
- Stale node detection (5-minute timeout)
- Node registration, update, and removal

**Infrastructure**
- Elasticsearch 8.11.4 with auto-created index templates
- Redis caching for search results (30s), suggestions (60s), directory (5min)
- NATS JetStream event streaming for teaser lifecycle events
- PostgreSQL via Prisma ORM for metadata storage

**Transparency Website**
- Next.js 14 public dashboard with 24 EU language support
- Statistics page with Recharts visualizations
- Real-time health monitoring (10s polling)
- Content type browser, publisher directory
- Dynamic legal pages (imprint, privacy, terms)

**Security**
- Helmet HTTP security headers
- Global rate limiting (120 req/min/IP)
- Per-publisher Redis sliding window rate limiter
- SHA-256 hashed API keys with constant-time comparison
- Input validation with class-validator (whitelist + forbidNonWhitelisted)
- Configurable CORS

**DevOps**
- Multi-stage Docker builds for API and Web
- Production Docker Compose stack with Caddy reverse proxy
- GitHub Actions CI pipeline (lint, build, Docker)
- Environment validation with startup checks
</Update>
