Every Discovery Node includes a public-facing **transparency website** — a Next.js 14 application that provides visibility into the node's operations, indexed content, registered publishers, and health status. This is a core part of roadbeat's commitment to transparency and openness.

## Overview

The website runs on port **3301** and is built with Next.js 14 (App Router), Tailwind CSS, shadcn/ui, Lucide React icons, and Recharts for data visualization. It supports **24 EU languages** via next-intl.

## Website API

The website communicates with the Discovery Node API via dedicated read-only endpoints at `/api/web/v1`. These are separate from the main API to allow independent rate limiting and caching.

| Endpoint | Description | Cache |
|----------|-------------|-------|
| `GET /api/web/v1/node-info` | Node identity and operator info | 5 min |
| `GET /api/web/v1/metrics` | Homepage metrics bar data | 1 min |
| `GET /api/web/v1/content-types` | Content types with document counts | 2 min |
| `GET /api/web/v1/publishers` | Publishers list (paginated, filterable) | 1 min |
| `GET /api/web/v1/health` | Component health status | No cache |
| `GET /api/web/v1/incidents` | Recent incidents | 30 sec |
| `GET /api/web/v1/legal/:page` | Legal page content (imprint/privacy/terms) | 1 hour |

<Callout kind="info">
  Website API endpoints are rate-limited to 120 requests/minute per IP, separate from the main API throttle.
</Callout>

## Internationalization

The website supports all **24 official EU languages**:

bg, cs, da, de, el, en, es, et, fi, fr, ga, hr, hu, it, lt, lv, mt, nl, pl, pt, ro, sk, sl, sv

Locale routing uses the `[locale]` App Router segment with `next-intl` v4. The locale prefix is used only when needed (English is the default). A language switcher in the header allows visitors to change languages.

All UI strings (~120 keys) are stored in JSON message files under `apps/web/src/messages/`.

## SEO

The website includes:
- **`sitemap.ts`** — dynamic sitemap generation for all pages and locales
- **`robots.ts`** — search engine crawling rules
- **Per-page Metadata** — title, description, and Open Graph tags for social sharing
- **Locale-aware canonical URLs** for all pages

## Production deployment

In production, the website is served behind **Caddy** as a reverse proxy with automatic HTTPS. The Docker production stack (`docker/docker-compose.prod.yml`) includes a dedicated web container built from `docker/web.Dockerfile` (multi-stage build).
